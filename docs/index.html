API checks from N. California, Montreal, Ireland, Stockholm, and Sydney:

overview

When we as developers want to provide a JSON API on the web, we choose from options for hosting providers, runtimes, and SQL databases. This project is my attempt to prototype those options and share what I learn.

Hosting providers:

  • Deno Deploy
  • Fly.io
  • Heroku
  • Railway
  • Render
  • Vercel

Runtimes (frameworks):

  • Deno (stdlib)
  • Go (stdlib)
  • Node (Express)
  • Ruby (Sinatra)

SQL databases:

  • Postgres
  • Supabase

Each stack has a healthcheck-style API endpoint that executes a SELECT 1 to a SQL database and responds with JSON { "status": "ok" }.

heroku + ruby + postgres

From 2009-2017 and 2020-2022, my go-to stack has been Heroku + Ruby + Postgres:

  • Ruby app in a GitHub repository
  • GitHub pull requests w/ Continuous Integration (CI)
  • Merge into main branch auto-deploys to Heroku staging environment
  • Manually promote staging to production environment in a pipeline
  • Postgres database configured with a follower

My favorite parts of Ruby are:

  • Expressive DSLs like Sinatra, RSpec, FactoryBot, and ActiveRecords migrations.
  • Iteration of arrays and hashes with the Enumerable module.

My least favorite parts are:

  • Code loading. I prefer Go and Deno's more explicit systems.
  • Dependencies. I prefer Go and Deno's standard libraries.
